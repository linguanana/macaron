We had a bug in the OAuth scope lookup flow. One service path was still reading scope data from the old S3 bucket after we migrated storage to MySQL. The issue was hard to catch because most paths were working fine — only one specific code path was hitting the old source. We found it by comparing latency metrics: some requests were taking 150-200ms while others were under 10ms. Once we identified the stale read path, we fixed the read logic and added a validation check to make sure all paths were consistent. Main lesson: during a storage migration, you have to audit every single read path, not just the main one. Feature flags helped us roll back safely while we fixed it.
