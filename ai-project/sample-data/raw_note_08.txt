We needed observability for the OAuth services at Zoom. Before dashboards, issues were found by users reporting problems. We set up metrics, logging, and PagerDuty alerts. Key SLIs we tracked: token request latency (p50, p95, p99), error rate by endpoint, and cache hit rate. We defined SLOs: 99.9% availability, p99 latency under 200ms. The most useful alert was on error rate spike — it caught two incidents before users noticed. Lesson: the most important metric is not the one that looks impressive on a dashboard. It is the one that fires an alert before a customer complains. Start with error rate and latency. Everything else is secondary.
