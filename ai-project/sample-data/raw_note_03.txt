We needed to automate OAuth scope updates across multiple services. Before this, whenever a scope changed, someone had to manually notify each team and they would update their service by hand. This took days and caused inconsistencies. We replaced it with a pub-sub pipeline using an internal Kafka-like event streaming platform. When a scope change happens, an event is published, and each downstream service consumes it and updates automatically. The whole propagation now takes minutes instead of days. Lesson: if a workflow requires manual coordination between multiple teams, it is a good candidate for event-driven automation. The hard part was not the code — it was convincing teams to trust the automated flow.
