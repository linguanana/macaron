Interview problem: binary search. I kept making off-by-one errors on the boundary conditions. The clean pattern is: left = 0, right = len(array) - 1, while left <= right, mid = left + (right - left) // 2. Use (right - left) // 2 instead of (left + right) // 2 to avoid integer overflow. When the condition is met, return mid. When target is less than mid, set right = mid - 1. When greater, set left = mid + 1. The off-by-one errors usually come from using right = mid instead of right = mid - 1. Write the invariant clearly before coding: at all times, the answer is within [left, right]. This mental model prevents most mistakes.
