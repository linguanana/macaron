At Amazon internship, I worked on a plugin for the Item Pipeline Workflow to publish merchant contribution entities via SNS. The challenge was making the plugin modular so other teams could use it without modifying core pipeline code. We used a plugin interface pattern — the core pipeline calls a standard method, and each plugin implements it differently. This meant teams could add new downstream consumers without touching shared code. Refactored three existing Java services to use the new plugin. Lesson: when building shared infrastructure, design for extension not modification. Open/closed principle matters a lot in large codebases where many teams share the same foundation.
