# Coding Framework

Before writing code, always explain:

1. what classes or functions you need
2. what fields each object should have
3. what data structure you will use
4. what the flow of the solution is

## Why This Matters

Many mistakes happen because the candidate starts coding before fully understanding:
- what the input means
- what state needs to be stored
- what each field represents
- what the rule actually is

## Common Coding Follow-ups

- What is the time and space complexity?
- What test cases would you write?
- What edge cases are easy to miss?
- What would change if the input became very large?
- What if this had to work across multiple servers?
