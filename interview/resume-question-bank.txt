John Resume Question Bank

Goal:
- Help him answer resume and general questions more clearly
- Keep answers short, structured, and not too rambling
- Add a few AI-related questions that fit a backend engineer, not an ML engineer


How he should answer

Good default structure for most questions:
1. Start with one-sentence summary
2. Explain 2 or 3 important details
3. End with impact or outcome

Good length:
- Usually 45 to 90 seconds
- Do not start from the beginning of his life
- Do not explain every tiny technical detail unless asked

Good pattern:
- Problem
- What he did
- Result

If he talks too much:
- Start with the headline first
- Cut extra background
- Give only the details needed to support the headline

4. Can you walk me through the S3 to MySQL migration?

What a good answer should sound like:
- old system
- why it was limiting

Example answer:
Originally the scope data lived in S3, which worked for simple storage but became limiting for query flexibility, consistency, and service behavior as the platform grew. I worked on moving that data into MySQL with a more structured schema, which improved access patterns and made the data model easier to manage in the services. We rolled it out in stages so we could validate behavior in production before fully switching over.

Example answer:
S3 was fine as basic object storage, but it was not a great fit for data that needed more structured access and stronger consistency in service logic. As the system evolved, query flexibility and update patterns became more important, and MySQL was a better fit for that kind of workload.

Quick coaching note for you

If he answers too long, interrupt with:
- What's the short version?
- Start with the headline first.
- Give me the 30-second version.
- What was the problem, what did you do, and what changed?

If he gets too technical too early, interrupt with:
- Explain it at a high level first.
- What is the most important point?
- What should I remember from that answer?
