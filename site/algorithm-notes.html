Algorithm Notes

Quick reminders for common patterns that keep showing up in backend-style coding questions.

HashMap

Use when you need fast lookup by key or state by user / token / session / request id.

Heap / Priority Queue

Use when you need the smallest or largest current item, top-k, or k-way merge.

Queue / Deque

Use when you need to remove old items from the front or maintain a time window.

Sliding Window

Use when the rule depends on recent events, time windows, or moving counts.

Doubly Linked List

Use when ordering changes often and you need O(1) insert / remove.

K-Way Merge

If multiple sources are already sorted, keep one candidate from each source in a heap instead of sorting everything again.
