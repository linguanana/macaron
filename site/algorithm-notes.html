Algorithm Notes

Quick reminders for common interview data structures, patterns, and mental triggers.

1. HashMap (The State Manager)

Use for fast lookup and state tracking by user, token, session, request id, or client id.

Key Functions: getOrDefault, putIfAbsent, containsKey.

Common Examples: Token Manager, Deduplicator, Counters.

2. Heap / Priority Queue (The Ranker)

Use when you need the smallest or largest current item, Top K, or merging sorted streams.

Key Functions: offer / poll, peek, custom comparator.

Common Examples: Log Merger, Top K Frequent Elements.

3. Queue / Deque (The Window Tracker)

Use when events are processed in order or when you need to remove old items from the front.

Key Functions: offerLast / pollFirst, peekFirst.

Common Examples: Rate Limiter, Retry Queue, Login Failure Tracker.

4. Sliding Window (The Event Filter)

Logic Pattern:

  1. add the new item
  2. while the condition is violated, remove old items
  3. update the result

Common Example: Login Failure Tracker.

5. Doubly Linked List + HashMap

Use when you need O(1) lookup and O(1) reordering.

Common Example: LRU Cache.

6. Exponential Backoff

Use when a background task fails and needs to be retried later without crashing the system.

Common Examples: Webhook retry queue, failed payment processing.

7. Distributed Lock

Use when multiple servers may try to update the same resource at the same time.

Common Examples: inventory update, wallet balance transaction.

8. Binary Tree (DFS / BFS)

Use when data is shaped like parent → left child / right child.

Common fields: value, left, right.

Key idea: a binary tree node has at most two children. This is different from a graph, where a node can have many neighbors.

DFS patterns:

  • Preorder: visit node, dfs(left), dfs(right)
  • Inorder: dfs(left), visit node, dfs(right)
  • Postorder: dfs(left), dfs(right), visit node

BFS pattern: put root in queue, pop front, visit node, add left and right children if they exist.

Common Examples: max depth, level order traversal, validate binary search tree, find a value.

9. Two Pointers (The Index Walker)

Use for sorted arrays, palindrome checks, pair problems, and merging two streams.

Common patterns: opposite ends moving inward, fast/slow pointers moving in the same direction, and two-input merge traversal.

Template: left = 0, right = n - 1, then move one or both pointers based on conditions.

10. Prefix Sum (The Range Sum Accelerator)

Use when you need repeated range-sum queries over an array.

Key idea: prefix[i] = nums[0] + ... + nums[i].

Subarray sum: sum of i..j is prefix[j] - prefix[i - 1] (if i > 0).

11. Linked List Core Moves

Use for pointer manipulation and in-place structural updates.

  • Insert after prev: node.next = prev.next; prev.next = node
  • Delete after prev: prev.next = prev.next.next
  • Fast/slow pointers: middle node, cycle detection, split-list tasks
  • Reverse list iteratively: track prev, curr, next

12. Monotonic Stack / Queue

Use for next greater/smaller element and window max/min patterns.

  • Increasing stack: pop while top >= current
  • Decreasing stack: pop while top < current
  • Amortized O(n): each element is pushed/popped at most once

13. Graph Essentials (DFS / BFS)

Use for connectivity, components, and shortest path in unweighted graphs.

Representations: edge list, adjacency list, adjacency matrix, and matrix-as-graph (grid problems).

Core rules: always track seen/visited; use BFS for shortest path in unweighted graphs.

14. Binary Search (Boundaries Matter)

Use on sorted search space or monotonic true/false conditions.

Standard: compare at mid, shrink left/right boundary, return insertion point if not found.

Duplicates: for leftmost target, shrink right on >=; for rightmost insertion, shrink right only on >.

15. Backtracking (Build + Undo)

Use to enumerate combinations, permutations, or subsets.

  1. Choose a candidate
  2. Recurse
  3. Undo the choice

16. Dynamic Programming (State + Transition)

Use when subproblems overlap and optimal substructure exists.

  • Define state meaning (dp[i] or dp[state])
  • Define recurrence (transition)
  • Define base cases
  • Pick top-down (memoization) or bottom-up (tabulation)

Example recurrence: Min Cost Climbing Stairs: dp[i] = min(dp[i-1] + cost[i-1], dp[i-2] + cost[i-2]).

Mental Triggers

  • Top K? → Heap
  • Recent X minutes? → Deque + Sliding Window
  • O(1) lookup? → HashMap
  • Already sorted? → K-Way Merge
  • Deduplication? → Request ID in a Map
  • Left and right child? → Binary Tree
  • Dependencies / neighbors / services? → Graph
