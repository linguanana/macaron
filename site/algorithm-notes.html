Algorithm Notes

Quick reminders for common interview data structures, patterns, and mental triggers.

1. HashMap (The State Manager)

Use for fast lookup and state tracking by user, token, session, request id, or client id.

Key Functions: getOrDefault, putIfAbsent, containsKey.

Common Examples: Token Manager, Deduplicator, Counters.

2. Heap / Priority Queue (The Ranker)

Use when you need the smallest or largest current item, Top K, or merging sorted streams.

Key Functions: offer / poll, peek, custom comparator.

Common Examples: Log Merger, Top K Frequent Elements.

3. Queue / Deque (The Window Tracker)

Use when events are processed in order or when you need to remove old items from the front.

Key Functions: offerLast / pollFirst, peekFirst.

Common Examples: Rate Limiter, Retry Queue, Login Failure Tracker.

4. Sliding Window (The Event Filter)

Logic Pattern:

  1. add the new item
  2. while the condition is violated, remove old items
  3. update the result

Common Example: Login Failure Tracker.

5. Doubly Linked List + HashMap

Use when you need O(1) lookup and O(1) reordering.

Common Example: LRU Cache.

Mental Triggers

  • Top K? → Heap
  • Recent X minutes? → Deque + Sliding Window
  • O(1) lookup? → HashMap
  • Already sorted? → K-Way Merge
  • Deduplication? → Request ID in a Map
