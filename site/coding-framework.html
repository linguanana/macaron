Coding Framework

Before writing code, always explain the shape of the solution first.

The Four Things To Say First

  1. what classes or functions you need
  2. what fields each object should have
  3. what data structure you will use
  4. what the flow of the solution is

Why This Matters

  • Many mistakes happen because coding starts before the problem is understood.
  • The candidate should be clear about what the input means and what state must be stored.
  • Practical backend questions go much better when the object model is defined first.

Common Coding Questions

  • Login Failure Tracker
  • Session Store
  • Token Manager
  • API Rate Limiter
  • Request Deduplicator
  • Merge Logs from N Servers
  • LRU Cache
