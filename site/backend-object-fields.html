Backend Object Fields

A simple cheat sheet for common backend objects and the fields they usually need.

LogEntry

  • timestamp
  • message
  • serverId

LoginTracker

  • userId
  • failed timestamps
  • maybe lock-related state

SessionRecord

  • sessionId
  • userId
  • expiryTime
  • invalidated / active

TokenRecord

  • token
  • userId
  • expiryTime
  • revoked

RateLimitRecord

  • userId or clientId
  • request timestamps
  • window size
  • limit

RequestRecord

  • requestId
  • payload hash
  • createdAt
  • status

CacheNode

  • key
  • value
  • prev
  • next

HeapEntry

  • timestamp or priority
  • source / server index
  • item index
  • object reference if needed
